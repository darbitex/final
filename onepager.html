What is Darbitex?

Darbitex is a fully decentralized exchange ecosystem on Aptos. At its core is a surplus-fee AMM: an immutable x × y = k pool paired with a composable routing layer that charges 10% only on measurable surplus over what a direct swap would have produced. No surplus, no fee.

Around this core sits a constellation of satellite packages — each is an independent on-chain module that composes the core's public primitives without requiring any upgrade to the core itself. Cross-venue aggregation, flash arbitrage, token creation, time-locked vaults, LP staking — all live on-chain with zero admin surface, zero servers, and zero backend infrastructure.

The frontend is hosted on Walrus (decentralized storage on Sui). There is no centralized server anywhere in the stack. Darbitex reads from and writes to Aptos mainnet directly.

The 10% Rule

Darbitex charges 10% only on measurable surplus: the delta between what the routed trade produced and what a direct canonical pool swap would have produced. Swap $1 and receive $1 — no charge. Swap $1 and receive $1.10 thanks to multi-hop routing — we take $0.01 (10% of the $0.10 improvement).

No baseline = no charge. If the direct pool doesn't exist, there is no reference point, so the entire output goes to the caller. Enforced in code, not convention.

Four Execution Surfaces

Each surface ships as an entry wrapper (for wallets), a composable public fun (FA-in/FA-out, for satellite packages), and a #[view] quote (for frontends).

Universal Flash Loans

Every pool supports flash loans at 1 bps. Flash receipts are hot-potato structs — no abilities, must be consumed by flash_repay in the same transaction or the tx aborts. close_triangle_flash uses this internally for zero-capital cycle closure.

LP as NFT

LP positions are Aptos objects. add_liquidity uses Uniswap V2 optimal-amount math — excess stays in your wallet. claim_lp_fees harvests accrued fees without touching principal. remove_liquidity burns the position for proportional reserves plus unclaimed fees. Positions are freely transferable, lockable, and stakeable.

Satellite Ecosystem — 6 Packages

Satellites are independent on-chain packages that compose the core's public primitives via cross-package use darbitex::pool / use darbitex::arbitrage. Adding a new feature never requires a core upgrade. Each satellite has its own audit trail, its own upgrade policy, and its own multisig governance.

Cross-Venue Aggregator compat

Compares quotes across Darbitex, Hyperion CLMM, ThalaSwap V2, and Cellana in parallel. Executes through whichever venue wins. Adapter satellites wrap each venue's native interface into primitive-only Move view/entry functions.

Flash Arbitrage (Flashbot) compat

Cross-venue flash arb: borrows from Aave V3 (0 fee), swaps through Darbitex + an external venue, repays, splits profit 90% caller / 10% treasury. Supports Thala, Hyperion, and Cellana routes. Anyone can trigger.

Token Factory frozen

Fire-and-forget token creator. Pick a name and symbol, pay a tiered fee (1-char: 1000 APT down to 5+: 0.1 APT), receive 1B supply of a new FA token. Self-burn via BurnCap. No future minting possible.

Token Vault frozen

Three modes for any FA token: Lock (time-based release), Vest (linear vesting schedule), Stake (deposit into reward pools, earn any token over time). Permissionless pool creation.

LP Staking compat

Stake Darbitex LP positions into reward pools. Earn any reward token over time while still collecting LP swap fees. Anyone can create a reward pool for any Darbitex pool + any reward token. Anyone can deposit rewards.

LP Locker compat

Wraps LP positions with a time-based unlock gate. LP fees remain claimable while locked — only the principal (the LP position itself) is gated by the unlock timestamp. Used for protocol-owned liquidity locks.

FROZEN = upgrade policy set to immutable (permanently locked on-chain, can never change). COMPAT = upgrade policy set to compatible (bug-fix runway during soak period, will flip to immutable after).

Audit — 13 Passes Across 3 Rounds

The core package went through 3 audit rounds with 13 independent AI auditor passes: Gemini 2.5 Pro, Grok 4, Qwen, Kimi K2, DeepSeek, ChatGPT GPT-5, Claude Opus 4.6, and Perplexity. 14 actionable fixes applied. All 5 R3 auditors returned GREEN.

Each satellite has its own audit trail (R1 minimum, 5 auditors each). The Token Vault and Token Factory are FROZEN (immutable on-chain) after passing audit.

Audits are aids, not guarantees. Read the code.

Governance

All packages are published from a 3-of-5 multisig. Treasury address and fee rates are hardcoded Move constants — changing them requires a package upgrade, which itself requires the 3-of-5 threshold. FROZEN satellites cannot be changed at all, ever.

Soak-period packages (COMPAT) will flip to immutable after 3–6 months of mainnet operation with no critical bugs.

When $DARBITEX?

The DARBITEX token has been created via the Token Factory satellite: 1 billion supply, 8 decimals. It is not yet in circulation.

Launch criteria: the token goes live when the protocol reaches self-sustainability:

1. POC proven. The core AMM, smart routing, flash arbitrage, cross-venue aggregation, token factory, vault, LP staking, and LP locker are all deployed and verified on Aptos mainnet. This milestone is complete.

2. Treasury bootstrap. When the treasury accumulates 100 APT from organic protocol fees (the 10% surplus charge + satellite creation fees), those funds bootstrap the initial DARBITEX/APT liquidity pool on Darbitex itself.

3. LP staking activation. Once the liquidity pool exists, LP staking reward pools go live with DARBITEX as the reward token. LP providers who stake earn DARBITEX. The protocol's own trading fees continuously refill the treasury, which continuously funds LP staking rewards.

At that point Darbitex becomes a self-sustaining decentralized protocol: trading fees fund the treasury, the treasury funds LP incentives, LP incentives attract liquidity, liquidity enables better routing and more surplus capture, which feeds back into the treasury. No external funding, no team allocation, no VC — just a flywheel powered by actual protocol usage.
