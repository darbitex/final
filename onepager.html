What is Darbitex Final?

Darbitex Final is a programmable arbitrage AMM on Aptos. It pairs an immutable x × y = k pool primitive with an opinionated routing layer that ships smart multi-hop swaps, real-capital cycle closure, and flash-loan triangle arbitrage as first-class on-chain operations. The protocol takes 10% of measurable surplus over the canonical direct baseline — and nothing when there is no baseline to improve on.

Final is a clean-slate successor to Darbitex Beta, not an upgrade. Beta's hook-NFT-at-birth system is removed. The pool layer is stripped to pure primitives and a new arbitrage module delivers the programmable routing surface. Beta remains LIVE at its own address; Final is an independent package.

The 10% Rule — Only on Measurable Surplus

Darbitex Final charges 10% only on measurable surplus: the delta between what a routed trade actually produced and what a direct canonical pool swap would have produced for the same input. If the user swaps $1 and receives $1, we take nothing. If the user swaps $1 and receives $1.10 thanks to smart routing, we take $0.01.

No baseline = no charge. If the canonical direct pool doesn't exist, there is no reference point, so the entire output goes to the caller. Darbitex refuses to tax trades where it is the only available path — a policy enforced in code, not convention. The guard lives in every composable surface: execute_path_compose, swap_compose, close_triangle_compose, and close_triangle_flash_compose.

The 1 bps swap fee on the underlying pool primitive goes 100% to LPs via the MasterChef V2 per-share accumulator. There is no passive protocol fee slot.

Four Execution Surfaces

The arbitrage module exposes four programmable execution surfaces, each available as an FA-in/FA-out composable primitive and as a user-facing entry wrapper:

SurfaceRole
execute_pathSmart multi-hop swap along a pre-computed optimal path
swapAuto-routed in→out swap; finds the best path internally
close_triangleReal-capital cycle closure — user provides seed capital to rebalance a triangle
close_triangle_flashFlash-borrow → cycle → repay, zero up-front capital

Path and cycle search use a bounded DFS with DFS_VISIT_BUDGET = 256, MAX_HOPS = 4, and MAX_CYCLE_LEN = 5 — enough to cover all realistic retail arbitrage routes while making gas DoS structurally impossible. Candidate pools are fetched via lazy pagination: one 10-pool page at a time, budget decremented per candidate, with the remaining budget safely threaded through nested cycle searches.

Three-Tier Composable API

Every arbitrage surface ships in three layers for maximum Move interoperability:

Tier 1 — Entry wrapper. public entry fun. Takes &signer, min_net_profit, deadline, and metadata. Withdraws from primary stores, runs the operation, deposits results. For direct wallet transactions.

Tier 2 — Compose (FA-in / FA-out). public fun. Takes a FungibleAsset, returns a FungibleAsset. No signer, no stores. Satellite packages compose this layer via cross-package use darbitex::arbitrage. The FA in the caller's hand is itself proof of authorization — Move's type system enforces it.

Tier 3 — View quote. #[view]. Simulates the operation against current state and returns the projected output + treasury cut. Frontend and off-chain routers read this directly from RPC.

Immutable Pool Primitive

pool.move is zero admin surface. No pause, no set_fee, no withdraw_protocol_fee, no upgrade hatch that can change pool state. Every operation — swap, add_liquidity, remove_liquidity, claim_lp_fees, flash_borrow, flash_repay — is free of any admin gate.

Swap math uses u256 intermediates. The k_after >= k_before invariant is enforced on every mutation including flash repay. Treasury cut arithmetic widens to u128 to rule out any overflow on whale-scale surpluses.

LP as NFT

LP positions are Aptos objects. add_liquidity uses Uniswap V2 router-style optimal amount computation — any slippage buffer you provide stays in your wallet if it isn't needed. claim_lp_fees harvests accrued fees without touching principal, and remove_liquidity burns the position for proportional reserves plus unclaimed fees in one shot. Positions are freely transferable.

Universal Flash Loans

Every pool supports flash loans at the same 1 bps rate. Flash receipts are hot-potato structs with no abilities — they must be consumed by flash_repay in the same transaction or the TX aborts. close_triangle_flash uses this internally to close rebalancing cycles with zero up-front capital.

Audit — 13 Passes Across 3 Rounds

Final went through 3 audit rounds with 13 independent AI auditor passes: Gemini 2.5 Pro (R1 + R3), Grok 4, Qwen, Kimi K2 (R1 + R3), DeepSeek (R1 + R3), ChatGPT GPT-5 (R1 + R2), fresh Claude Opus 4.6 web (R2 + R3), in-session Claude, and Perplexity. A total of 14 actionable fixes were applied across the rounds:

Round 1 (R1 + R1.5): 8 fixes including LP fee accumulator tightening, flash repay excess donation policy, dead E_SAME_TOKEN removal, doc comment placement, and canonicalization of the swap entrypoint against baseline manipulation.

Round 2 (R2.1 + R2.2): 3 fixes. R2.1 HIGH — Gemini caught a zero-baseline tax bug in swap_compose; fixed by mirroring the "no baseline = no charge" guard from execute_path_compose. R2.2 HIGH — DoS vector via fetch_all_sister_pools; fixed by inlining lazy-paginated DFS with per-candidate budget decrement in both dfs_path and dfs_cycle.

Round 3 (R3.1): 3 fixes. Claude R3 caught that the outer loop of find_best_flash_triangle didn't decrement the shared budget per candidate; fixed by threading &mut budget through. Kimi R3 caught that treasury-cut math could overflow u64 on whale surpluses; fixed by widening to u128. Stale comment cleanup in pool.

Final verdict: all 5 R3 auditors returned 🟢 GREEN. Gemini — the most adversarial auditor throughout — concluded: "The architecture is clean, the fallback logic is flawless, and the protection mechanisms against both economic manipulation and execution-halting DoS are mathematically sound. Proceed to publish."

Audits are aids, not guarantees. Read the code.

Fully Decentralized

Darbitex Final has no servers:

Smart contracts — on Aptos mainnet, published from a 3-of-5 multisig. upgrade_policy = "compatible" during a 3–6 month soak, then flipped to immutable.

Frontend — to be hosted on Walrus (decentralized storage on Sui).

Backend — there is none. All state lives on-chain; pools, reserves, routes, and quotes are read directly from Aptos RPC.

Governance

Final is published from a 3-of-5 multisig at the same address as the package (Aptos unifies publisher and package). Owners 1–4 are preserved from Beta; owner 5 is a new hot wallet replacing the frozen Beta alpha wallet.

TREASURY is a hardcoded Move constant identical to the Beta treasury multisig. Changing it would require a package upgrade, which itself requires the 3-of-5 threshold. TREASURY_BPS = 1_000 (10%) is also hardcoded. Minimum-trust by design.

For Builders

Pool creators — call pool_factory::create_canonical_pool(creator, meta_a, meta_b, amount_a, amount_b). Permissionless. You receive the initial LP position.

Aggregators — enumerate via pool_factory::get_all_pools(), quote via arbitrage::view_* functions, and compose via the Tier-2 public fun primitives: arbitrage::swap_compose(), execute_path_compose(), close_triangle_compose(), close_triangle_flash_compose().

LPs — pool::add_liquidity() with auto-optimal buffering, pool::claim_lp_fees() without withdrawing, pool::remove_liquidity() with slippage floors.

Satellite packages — Final's three-tier composability means future features (meta-routers, farming, external-venue bridges) ship as independent packages depending on Final core via cross-package use darbitex::arbitrage. No core upgrade needed.
