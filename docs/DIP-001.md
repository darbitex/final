# DIP-001: Darbitex Organic Evolution — From AMM to Autonomous Yield Engine

**Status**: Draft
**Proposer**: Antigravity (AI Operator)
**Date**: 2026-04-20

## Abstract
This proposal outlines the strategic roadmap for Darbitex beyond the R4 deployment. It focuses on transforming Darbitex from a static liquidity provider into an autonomous, MEV-capturing infrastructure that maximizes organic growth through three pillars: Keeper Decentralization, MEV-Yield Integration, and Oracle Monetization.

## Pillar 1: Incentivized Keeper Network (The Decentralization Path)
Currently, TWAMM execution relies on a whitelisted keeper.
- **Proposal**: Implement a permissionless keeper registration backed by DARBITEX token staking.
- **Mechanism**: Keepers stake DARBITEX to earn a portion of the "Keeper Fee" from virtual orders. Slashing conditions apply for downtime or missed executions.
- **Goal**: Increase liveness and create a structural buy-pressure for the DARBITEX token.

## Pillar 2: MEV-Yield Injection (The Liquidity Magnet)
Leverage the internal MEV capture from the Flashbot/Bridge.
- **Proposal**: Direct 5-10% of Flash-Arb profits directly into the `lp-staking` satellite.
- **Mechanism**: Automatically convert arb profits into LP rewards for active stakers.
- **Goal**: Provide LPs with "MEV Dividends," making Darbitex the highest-yielding AMM for core pairs in the Aptos ecosystem.

## Pillar 3: Oracle-as-a-Service (The Ecosystem Multiplier)
The TWAMM EMA Oracle is a high-integrity data source based on real reserves.
- **Proposal**: Standardize and expose the TWAMM EMA as a public oracle interface for other Aptos lending protocols.
- **Mechanism**: Implement a subscription or fee-per-query model for external protocols accessing Darbitex price data.
- **Goal**: Establish Darbitex as the source-of-truth for asset pricing, driving more integrations and secondary volume.
